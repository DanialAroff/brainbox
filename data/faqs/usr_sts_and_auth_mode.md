USR_STS and AUTH_MODE

USR_STS can have
S- Suspended
A- Active
I - Inactive
N- New
E - Expired
RCe did not have 'N' value because after register normally user already login for the first time.. The other customer still have N value for USR_STS
 
AUTH_MODE can have
S- SMS
E- Email
Authmode depend on customer.. RCE sof ar only use 'S' which refer to SMS for authentication mode to receive OTP/TAC value
